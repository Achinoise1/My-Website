## ...state

摘自Wetab：在 JavaScript 中，当一个对象不再被引用时，它会被标记为可回收的，并且在合适的时机由 JavaScript 的垃圾回收机制自动进行回收。在这种情况下，如果旧的 state 对象没有其他引用指向它，那么它会被标记为可回收的，并在垃圾回收机制的下一个运行周期中被自动回收。

由于使用展开运算符 `...state` 创建了一个新的对象，并对其进行了修改或添加属性，原始的旧 state 对象会在没有引用指向它时变得不可访问。一旦旧的 state 对象没有任何引用指向它，JavaScript 引擎会在合适的时机将其回收。

需要注意的是，自动回收的时机是由 JavaScript 引擎决定的，具体的回收时间可能会受到多种因素的影响。一般来说，理论上当旧的 state 对象不再被引用时，会在某个时间点被垃圾回收机制回收，释放相应的内存空间。
